# 鸿蒙 APP 构建与测试分发指南（包名：com.filescanner.app）

本指南覆盖：安装新增依赖 → 编译 HAP → 把安装包发给测试用户（纯血 HarmonyOS NEXT 手机）。

---

## 0. 环境前提（重要）
- DevEco Studio 6.1 已安装，工程 `D:\Harmony\HarmonyCleaner` 可编译 HAP。
- **本机没有独立 ohpm 命令行工具**：DevEco 把 ohpm 内置在 IDE 内，只能在 DevEco 的
  **Terminal 面板**（View > Tool Windows > Terminal）或 IDE 的 **Sync** 功能里使用。
- 因此下面所有命令请在 **DevEco 内置 Terminal** 中执行，不要在本机 PowerShell 执行。

---

## 1. 安装 pinyin-pro 依赖（拼音/繁简功能需要）
工程根 `oh-package.json5` 已加入依赖 `"pinyin-pro": "^3.0.0"`。

在 DevEco Terminal 执行（二选一）：
```
ohpm install
```
或只装这一个包：
```
ohpm install pinyin-pro
```
> 若默认源（ohpm.openharmony.cn）拉取失败，改用 npm 源：
> `ohpm install pinyin-pro --registry https://registry.npmjs.org/`

安装完成后，`entry/oh_modules/pinyin-pro` 应存在。

### 关于繁简与拼音
- **拼音**：由 `pinyin-pro` 生成（格式 `全拼 空格分隔|首字母`，如 `dou po cang qiong|dpcq`），
  搜索时输入 `dpcq` / `dou` / `cang` 都能命中。pinyin-pro 对繁体字同样能出拼音，所以繁体文件名的拼音搜索也有效。
- **繁简**：`pinyin-pro` 的 `convert` 只做“拼音格式转换”，不支持汉字繁简。本工程用 `utils/ChineseConverter.ts`
  内置的精简繁→简映射表（覆盖网文/小说文件名高频繁体字）。如需完整繁简，后续可接入 opencc-js 等库。

---

## 2. 编译模式说明（strictMode）
`entry/build-profile.json5` 已将 `buildOption.arkOptions.strictMode` 设为 `false`，
用于兼容 `pinyin-pro` 这类未做 ArkTS 严格适配的第三方库。**不要改回 true**，否则编译报严格模式错误。

---

## 3. 编译 HAP
DevEco Terminal：
```
hvigorw assembleHap --mode module -p product=default -p buildMode=debug
```
或菜单 Build > Build HAP(s)/APP(s) > Build HAP。

产物位置：
- 未签名：`entry/build/outputs/default/entry-default-unsigned.hap`
- 已签名：`entry/build/outputs/default/entry-default-signed.hap`

---

## 4. 给测试用户（其他纯血鸿蒙手机）安装

### 4.1 自动签名（仅限你自己的设备，不能给别人装）
DevEco 的 File > Project Structure > Signing Configs 勾选 “Automatically generate signature”
只在**你本人在 DevEco 登录并授权的设备**上有效。测试用户无法安装。

### 4.2 hdc + UDID 直装（适合少量测试机，每台需登记）
1. 测试机：设置 > 关于手机 > 连续点版本号开启开发者模式；系统和更新 > 开发人员选项 > 开启“USB 调试”。
2. 用 USB 连电脑，在 DevEco Terminal 取 UDID：
   ```
   hdc shell bm get --udid
   ```
3. 登录 [AppGallery Connect](https://developer.huawei.com/consumer/cn/agconnect/) → 用户与访问 →
   设备管理，添加设备（填上面的 UDID）。
4. 申请 **调试证书(.cer)** 与 **调试 Profile(.p7b)**（Profile 里包含这些设备），并本地用 keytool 生成密钥库(.p12)。
5. 在 build-profile.json5 的 `signingConfigs` 填入这些文件与密码（见第 5 节）。
6. Build HAP（debug），然后：
   ```
   hdc install entry/build/outputs/default/entry-default-signed.hap
   ```
   每台测试机都要重复 2–6（UDID 不同）。

### 4.3 AppGallery Connect 内部测试（推荐给多位测试用户）
1. 登录 AppGallery Connect → 我的项目 → 创建应用，包名 `com.filescanner.app`。
2. 菜单“应用服务 > 内部测试”创建测试，上传构建出的 **APP 包（.app）** 或 HAP。
3. 添加测试用户（其华为账号），系统发送邀请。
4. 测试用户点邀请链接/邮件即可安装，无需逐台登记 UDID。
> 构建 APP 包：`hvigorw assembleApp` 或用菜单 Build > Build APP(s)。

## 5. 配置真实签名（build-profile.json5 模板）
在 `build-profile.json5` 的 `app` 节点加入：
```json5
"app": {
  "signingConfigs": [
    {
      "name": "release",
      "type": "HarmonyOS",
      "material": {
        "certpath": "D:/Harmony/certs/upload_cert.cer",
        "storePassword": "你的密钥库密码",
        "keyAlias": "key0",
        "keyPassword": "你的密钥密码",
        "profile": "D:/Harmony/certs/upload_profile.p7b",
        "signAlg": "SHA256withECDSA",
        "storeFile": "D:/Harmony/certs/upload_keystore.p12"
      }
    }
  ]
}
```
证书/密钥库在 AppGallery Connect 申请，或用 DevEco 的 Generate Key and CSR 本地生成。

## 6. 常见问题
- **编译报 “Property 'convert' does not exist” 之类**：本工程已不使用 pinyin-pro 的 convert 做繁简，
  若仍报，检查 `utils/ChineseConverter.ts` 的 import 是否只引入了 `pinyin`。
- **拼音搜索不命中**：确认 `scanned_file` 表有 `title_pinyin`/`author_pinyin` 两列
  （旧库首次启动会自动 ALTER 补列），且扫描时已生成（ScanService 调用 `ChineseConverter.toPinyin`）。
- **繁体文件名未转简**：`ChineseConverter.toSimplified` 用内置精简表，极少数生僻繁体字可能未覆盖，属已知限制。
