# 文件清理助手 · PC 网页版 安装部署手册

网页版 = 后端（FastAPI + MySQL）+ 前端（静态 HTML/CSS/JS），通过浏览器访问。适用于内网 / 服务器统一部署、多人访问。

## 1. 环境要求

| 项目 | 要求 |
| --- | --- |
| 操作系统 | Windows / Linux / macOS（部署机） |
| Python | 3.8+ |
| 数据库 | MySQL 5.7+ / 8.0（外部，需自行安装） |
| 其他 | 能访问 MySQL 的账号；开放 8000 端口（或自定义） |

## 2. 获取代码与依赖

```bash
# 进入工程根目录
cd <工程根>/txt文件清理-单工程清理

# 安装 Python 依赖（也可由 start.py 自动安装）
pip install -r requirements.txt
```

依赖含：`fastapi`、`uvicorn`、`sqlalchemy`、`pymysql`、`cryptography`、`requests`、`python-multipart`、`chardet`、`openpyxl`、`python-dotenv`。

## 3. 配置 MySQL

后端通过**环境变量**读取 MySQL 连接（不硬编码密码）：

| 变量 | 说明 | 默认 |
| --- | --- | --- |
| `DB_USER` | MySQL 用户名 | **必填** |
| `DB_PASS` | MySQL 密码 | **必填** |
| `DB_HOST` | MySQL 主机 | `localhost` |
| `DB_PORT` | MySQL 端口 | 见 `backend/app.py`（常见 3306，本工程默认 3308） |
| `DB_NAME` | 库名 | `file_scanner_noai` |
| `CORS_ORIGINS` | 允许的前端来源 | `http://localhost:8000,http://127.0.0.1:8000` |

后端启动时会**自动创建数据库** `file_scanner_noai`（不存在时），并建表、补注释迁移。只需确保账号有建库 / 建表权限。

Linux / macOS 设置环境变量：

```bash
export DB_USER=scanner
export DB_PASS='你的密码'
export DB_HOST=127.0.0.1
# Windows PowerShell
$env:DB_USER="scanner"; $env:DB_PASS="你的密码"
```

## 4. 启动服务

### 方式一：一键脚本（推荐）

```bash
python start.py
```

脚本会：① 检查并自动 `pip install` 缺失依赖；② 启动后端 `python -m backend.app`；③ 等待服务就绪（最多 30 秒）；④ 打印访问地址。

### 方式二：手动启动

```bash
python -m backend.app
```

后端默认监听 **`0.0.0.0:8000`**（代码 `uvicorn.run(host='0.0.0.0', port=8000)`），前端由后端同源提供（`/` 返回 `frontend/index.html`，`/static` 提供 CSS/JS）。**无需再单独起静态服务器。**

## 5. 访问

浏览器打开：`http://localhost:8000`（本机）或 `http://<服务器IP>:8000`（局域网）。

> 安全提示：后端默认绑定 `0.0.0.0`，会监听所有网卡。若部署在公网 / 不可信网络，请务必用防火墙限制来源，或通过 Nginx 反向代理 + 鉴权暴露，不要裸奔。

## 6. 生产部署建议（可选）

- 用 `uvicorn` 多 worker 启动，前面架 **Nginx** 反向代理（HTTPS、静态缓存、限流）。
- 用 `systemd`（Linux）或 Windows 服务把后端跑成后台服务，日志重定向到文件。
- 示例（Linux systemd 片段）：

```ini
[Service]
Environment=DB_USER=scanner
Environment=DB_PASS=******
ExecStart=/opt/venv/bin/uvicorn backend.app:app --host 127.0.0.1 --port 8000 --workers 2
```

## 7. 关闭

- 一键脚本 / 手动前台运行：在终端按 `Ctrl+C`。
- 服务方式：停止对应服务 / 进程。
