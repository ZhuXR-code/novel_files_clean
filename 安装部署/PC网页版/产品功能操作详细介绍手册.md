# 文件清理助手 · PC 网页版 产品功能操作详细介绍手册

网页版与 APP 端共用同一套产品能力，区别只在「形态」：网页版在浏览器里操作，左侧是「扫描配置」列表，主区域是文件 / 合集浏览。

## 1. 整体布局

- **左侧栏「扫描配置」**：管理多个扫描配置（文件夹、文件类型、递归、排除目录）。点配置即可开始一次扫描，结果形成一个「文库」。
- **主区域「文库内容」**：顶部有「列表 / 合集」切换，以及筛选、排序、搜索；底部是分页导航条。
- **右上角菜单**：合集模式下有「勾选重复」「合集设置」。

## 2. 扫描配置

- 新增配置：名称（可选）、扫描文件夹、文件类型（如 `txt,md`）、是否递归、排除文件夹。
- 从配置一键「开始扫描 / 重新扫描」。扫描会递归读取文件、解析书名作者、计算内容指纹（SHA-256）用于识别重复。

## 3. 文库内容：列表模式

- 顶部「全部 / 已标记」筛选；按时间 / 名称 / 大小排序（点列头可切升 / 降序）。
- 关键字搜索框实时过滤。
- 每行可勾选（待删除）、加星标（标记）、点开查看明细。

## 4. 文库内容：合集模式

- 按「书名」把同名文件聚合成合集，便于发现重复。
- 「合集设置」可设：合集内文件数量区间（最少 ~ 最多，留空不限）、排除的合集书名。
- 合集可整体勾选 / 展开逐文件操作。

## 5. 分页导航条（大库浏览）

对齐 APP 端逻辑：

- **每页条数**：输入框（默认 100，范围 10~2000）填完 `onchange` 即生效（超出自动夹到 10~2000）。
- **跳至页**：输入框填目标页码，回车或点「跳转」直达。
- **首页 / 上一页 / 下一页 / 末页**：快速翻页。
- 页码信息显示「第 X / Y 页，共 N 条」。

> 这样面对十万级数据也只查一页，浏览流畅，不一次性拉全量。

## 6. 勾选重复

合集模式下「勾选重复」：在**同一合集内、按 (作者+小说名) 子分组**套用五则规则智能勾选待删重复项。算法见 `backend/dup_logic.py`，与桌面版、APP 端完全一致（Kotlin 端镜像同一逻辑）。也可「按书名/作者相同标记」批量加星标、「清除标记」取消。

### 6.1 五则规则

| 规则 | 名称 | 判定 |
| --- | --- | --- |
| ① | 完全相等去重 | `文件名 + 大小 + 小说名 + 作者 + 进度` 五字段完全一致的多本中，保留最新(创建最晚，并列取 id 最大) 一本，**其余全部勾选**。 |
| ② | 纯数字进度对比 | 同 (作者+小说名) 内，若**所有**进度均为纯数字(可含小数、尾随 `%`)，则**进度数字最大者不勾选**，其余纯数字文件全部勾选。 |
| ③ | 含中文进度 / 完结特例 | 进度含中文(如「完结/连载/断更」) 的**不勾选**；但若同组存在文件名带「完结/完本/全本/全集/完整/全套/全集版」等关键词、且「进度数字最大文件」的大小 **小于** 同组**所有**含中文进度文件的大小，则该「进度数字最大文件」**也要勾选**（存在更完整的完结版，部分进度版冗余应删）。 |
| ④ | 最大文件不勾选原则 | 已勾选文件中，本 (作者+小说名) 组内**唯一**文件大小最大者**不勾选**；若多本并列最大，则按大小无法区分、不据此保护，以免同尺寸重复组被整体保留。 |
| ⑤ | 完结+N番外 组合排序 | 进度【严格】匹配 `完结+数字番外`（如「完结+3番外」）的文件，在同 `(作者+小说名)` 组内按数字 N 排序：**数字最大者不勾选**，其余勾选（强制勾选，覆盖规则③A 对中文进度的保护）；但被勾选的文件若恰为本组 `(作者+小说名)` 内**文件大小最大者**，则也不勾选。 |

> 最终勾选集 = (规则①/② 勾选集 − 规则①/③/④ 保护集) ∪ 规则①/③ 强制勾选集。
> 规则①的精确重复(=同尺寸)不可能同时是「唯一最大文件」，故强制勾选不会与规则④冲突。

### 6.2 举例

**例一（规则① 完全相等去重）**
同一合集内《斗破苍穹》作者「天蚕土豆」有 3 个文件，名称/大小/进度完全一致，创建时间分别为 A(最早)、B、C(最新)：
- 结果：C 不勾选，A、B 勾选（删旧留新）。

**例二（规则② 纯数字进度对比）**
同书 3 个文件进度分别为 `12%`、`30%`、`30%`(都是纯数字)：
- 结果：两个 `30%` 并列最大 → 不勾选；`12%` 勾选（删低进度、留高进度）。
- 若进度为 `12%`、`30%`、`50%`：则 `50%` 不勾选，`12%`、`30%` 勾选。

**例三（规则③ 含中文进度保护 + 完结特例）**

  - 完结+N番外 组合（规则⑤）：F1 `完结+3番外`(2.0MB)、F2 `完结+5番外`(5.0MB)、F3 `完结`(6.0MB) → M={F1,F2} 按 N 排序，数字最大 F2 不勾选，F1 勾选（除非 F1 恰为本组最大文件）；F3 含中文受规则③保护不勾选。最终 F1 勾选、F2/F3 保留。
同书 3 个文件：
- F1：文件名 `斗破苍穹.txt`，进度 `80%`，大小 2.0 MB
- F2：文件名 `斗破苍穹完结版.txt`，进度 `完结`，大小 5.0 MB
- F3：文件名 `斗破苍穹.txt`，进度 `完结`，大小 6.0 MB

分析：进度含中文的 F2/F3 受规则③A 保护不勾选；进度最大的 F1(80%) 比所有含中文文件(F2/F3) 都小，且组内有「完结」关键词文件名 → 触发规则③B，F1 **强制勾选**。
- 结果：F1 勾选(删)，F2、F3 不勾选(保留更完整的完结版)。

**例四（规则④ 最大文件保护）**
同书 4 个文件，进度均为纯数字 `50%`、`50%`、`50%`、`50%`，大小分别为 1.0 / 2.0 / 2.0 / 4.0 MB：
- 规则②：进度并列最大 → 全部不勾选（无低进度可删）。
- 规则④：组内唯一最大为 4.0 MB 那本 → 即便被勾选也不删(保护)。
- 结果：本例无纯数字低进度项，最终**不勾选任何文件**。

**例五（规则④ 并列最大不保护）**
同书 3 个文件，进度 `10%`、`10%`、`10%`，大小均为 3.0 MB(完全相等)：
- 规则①：五字段完全相等 → 非最新 2 本勾选、最新 1 本不勾选。
- 规则④：最大尺寸 3.0 MB 有 3 本并列 → 不据此保护。
- 结果：非最新的 2 本勾选(删)，最新 1 本不勾选。

## 7. 选择并删除

勾选文件（或经「勾选重复」自动勾选）后，底部出现「删除选中」；确认后逐条删除并实时显示进度与成功 / 失败数。删除通过系统授权进行，确认页可反选。

## 8. 设置

- **主题 / 字号**：调整界面外观（网页版跟随浏览器 / 系统）。
- **关键词替换（文件名数据清洗）**：维护「查找 → 替换」（留空=删除）规则，分扫描阶段（文件名）与解析阶段（书名/作者/进度/来源）作用，按顺序执行。
- **导出已标记清单** / **清空本地数据**：备份与重置。

## 9. 帮助

界面内「帮助」提供上述流程说明，随版本更新。

## 10. 调试日志面板（操作留痕与一键复制）

界面内的「调试日志」面板（与帮助并列）提供两类日志，便于自排查与贴给运维：

- **操作日志**（`/api/operation-logs`）：用户关键操作留痕——勾选重复、删除记录（含文件）、清空配置、一键清理删除等，每条含时间、摘要与统计（如请求数 / 成功数 / 失败数）。对应落盘文件 `logs/operation.log`。
- **调试日志**（`/api/pipeline-logs`）：最近一次「一键清理」的流水线运行日志（结构化的步骤 / 耗时记录；未运行过一键清理时为空）。

两个 Tab 均可「一键复制」当前全部文本。
