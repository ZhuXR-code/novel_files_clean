# 文件清理助手 · PC 网页版 运维手册

## 1. 基本信息

- 服务名：文件清理助手（网页版）后端 FastAPI 应用 `backend.app:app`
- 默认端口：**8000**（绑定 `0.0.0.0`）
- 数据库：**MySQL**，默认库名 `file_scanner_noai`
- 环境变量：`DB_USER` / `DB_PASS` / `DB_HOST` / `DB_PORT` / `DB_NAME` / `CORS_ORIGINS`（详见安装部署手册）

## 2. 日志

- `python start.py`：后端日志实时打印到当前终端；出错时（含 `ERROR`/`Error`）会列出关键行。
- 服务方式：把 stdout/stderr 重定向到文件，例如 `uvicorn ... >> /var/log/filescanner.log 2>&1`。
- 后端使用 Python `logging`，关键词搜索：`ERROR`、`Warning`、`数据库`。

### 2.1 两类日志

| 类别 | 文件 / 接口 | 内容 | 用途 |
| --- | --- | --- | --- |
| 流水线调试日志 | 接口 `GET /api/pipeline-logs` | 最近一次「一键清理」的流水线运行日志（结构化的步骤 / 耗时记录，取自 `ParseLog`；未运行过一键清理时返回空） | 排查一键清理过程、节点耗时 |
| 操作日志（用户留痕） | 落盘 `logs/operation.log`；接口 `GET /api/operation-logs` | 标记重复、删除记录（含文件）、清空配置、一键清理删除等**用户关键操作** | 审计「谁删了什么」、复现操作链路 |

- `logs/operation.log`：由 `backend/operation_log.py` 写入，单文件 + 滚动切分（5MB × 10 备份），每条以 `[YYYY-MM-DD HH:MM:SS] [操作] ...` 开头。
- **前端「调试日志」面板**（界面底部 / 设置附近）：含「操作日志」「调试日志」两个 Tab，分别对应上面两个接口，并支持「一键复制」全部内容，便于贴给运维排查。

## 3. 数据库维护（MySQL）

备份：

```bash
mysqldump -u<用户> -p <DB_NAME> > fs_backup.sql
```

恢复：

```bash
mysql -u<用户> -p <DB_NAME> < fs_backup.sql
```

主要表（与 APP 端一致）：

- `scan_run`：每次扫描（文库）记录。
- `scanned_file`：解析后的文件明细。
- `scan_config`：扫描配置。
- `keyword_replace_rules`：关键词替换（文件名清洗）规则。

> 表结构以 `backend/models.py` 与 `backend/app.py` 的自动迁移为准；不要手工大改结构，升级时由代码自动迁移。

## 4. 升级

```bash
git pull            # 或拷贝新代码
pip install -r requirements.txt
# 重启服务
systemctl restart filescanner     # 或停止后重新 python -m backend.app
```

后端启动会自动执行库 / 表 / 注释迁移，通常无需手工操作。

## 5. 常见故障

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| 启动报数据库连接失败 | `DB_USER/DB_PASS` 未设置或错误 | 设置环境变量后重试 |
| `Access denied for user` | MySQL 账号无权限 | 给账号授权建库 / 建表权限 |
| 端口 8000 被占用 | 其他程序占了 8000 | 释放端口，或修改 `backend/app.py` 的 `uvicorn.run(port=...)` |
| 页面能开但接口 403/跨域 | `CORS_ORIGINS` 不含当前来源 | 把访问来源加入 `CORS_ORIGINS`（逗号分隔） |
| 解析 / 扫描卡住 | 大库写入超时 | 后端已对批量写入做重试与超时下调；检查 MySQL 负载 |

## 6. 安全与容量

- 后端默认绑定 `0.0.0.0`，**请勿在不可信网络裸奔**：用防火墙 / 反向代理 + 鉴权。
- `CORS_ORIGINS` 已收紧为指定来源，不要设成完全开放。
- 网页版所有数据落在 MySQL；定期 `mysqldump` 备份即可。
- 大库（十万级）已做分页查询，前端「每页条数」默认 100、范围 10~2000，「跳至页」可直达。
