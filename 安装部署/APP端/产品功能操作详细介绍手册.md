# 文件清理助手 · APP 端（Android）产品功能操作详细介绍手册

本文按界面逐一介绍功能，便于培训与答疑。

## 1. 首页（Home）

- **文件总数 / 已标记**：统计本机已扫描文件与已标记数量。
- **开始扫描**：进入「扫描配置」列表（见第 8 节），新增或选择配置后开始一次扫描，结果形成一个「文库」；扫描进行中可在「扫描进度」页 **停止扫描**，停止成功或扫描结束后出现 **重新扫描** 按钮，复用同一配置直接再次扫描。
- **查看文库**：进入文库列表（需已有扫描数据）。
- 底部提示「本地扫描，文件不会上传」强调纯本地、隐私安全。

## 2. 文库列表（每次扫描）

一次扫描 = 一个文库。列表展示：文库名称、来源文件夹、文件数、扫描时间。
- 点击文库 → 进入该文库内容（第 3 节）。
- 右侧删除图标 → 删除整个文库及其全部文件（不可恢复，有二次确认）。

## 3. 文库内容（Library）

顶部有「列表 / 合集」分段开关：

- **列表模式**：平铺全部文件，支持「全部 / 已标记」筛选、按时间/名称/大小排序、关键字搜索。
- **合集模式**：按「书名」聚合相同文件，便于发现重复（见第 4 节）。

底部「分页导航条」（详见第 5 节）用于大库分页浏览。

### 3.1 文件行操作

- 勾选框：加入「待删除 / 待处理」选择集。
- 星标：标记 / 取消标记（贯穿筛选与导出）。
- 点击整行：进入「文件明细」。

### 3.2 文件明细

展示解析出的全部字段：书名、作者、进度、来源、原文件名、扩展名、大小、是否标记、内容哈希、路径；并提供「打开文件」（调用系统查看器打开）。

## 4. 合集模式与合集设置

- 合集按书名把同名文件聚成一组；组内可整体勾选 / 取消，展开后逐文件操作。
- **合集设置**（右上角菜单 → 合集设置）：
  - 文件数量区间「最少 ~ 最多」：只显示组内文件数落在区间的合集，留空表示不限。
  - 排除的合集（书名）：一行一个或逗号分隔，命中即隐藏。
  - 确认 / 取消按钮已统一高度与样式，确保对齐。

## 5. 分页导航条（大库浏览）

位于文库内容底部，针对几万~十几万条设计：

1. **每页条数**：输入框（默认 100，范围 10~2000）填完点「应用」生效——不再提供 100/500/2000 预设按钮，改为自由输入更贴合电脑端习惯。
2. **跳到**：输入框填目标页码，点「跳转」直达该页（对齐电脑端「跳至页」）。
3. **首页 / 上一页 / 下一页 / 末页**：快速翻页，越界自动禁用。

> 改造说明：原「跳转」与预设按钮挤在同一行，窄屏下会被挤压导致输入不可用；现「每页」「跳转」各自独占一行，手机宽度下始终可用。另已修复分页输入框文字被高度裁剪、打字不可见的问题，现在输入内容与光标均正常显示。

列表过长时，屏幕右下角出现「回到顶层 / 回到底层」悬浮按钮，快速滚动。

## 6. 勾选重复

合集模式下，右上角菜单：
- **勾选重复**：与电脑端 / 网页版逻辑完全一致（共用算法 `backend/dup_logic.py`，Kotlin 端镜像同一逻辑），在「同一文库内、以 (作者+小说名) 子分组」套用以下五则规则，自动勾选待删重复项：

| 规则 | 名称 | 判定 |
| --- | --- | --- |
| ① | 完全相等去重 | `文件名 + 大小 + 小说名 + 作者 + 进度` 五字段完全一致的多本中，保留最新(创建最晚，并列取 id 最大) 一本，**其余全部勾选**。 |
| ② | 纯数字进度对比 | 同 (作者+小说名) 内，若**所有**进度均为纯数字(可含小数、尾随 `%`)，则**进度数字最大者不勾选**，其余纯数字文件全部勾选。 |
| ③ | 含中文进度 / 完结特例 | 进度含中文(如「完结/连载/断更」) 的**不勾选**；但若同组存在文件名带「完结/完本/全本/全集/完整/全套/全集版」等关键词、且「进度数字最大文件」的大小 **小于** 同组**所有**含中文进度文件的大小，则该「进度数字最大文件」**也要勾选**（存在更完整的完结版，部分进度版冗余应删）。 |
| ④ | 最大文件不勾选原则 | 已勾选文件中，本 (作者+小说名) 组内**唯一**文件大小最大者**不勾选**；若多本并列最大，则按大小无法区分、不据此保护，以免同尺寸重复组被整体保留。 |
| ⑤ | 完结+N番外 组合排序 | 进度【严格】匹配 `完结+数字番外`（如「完结+3番外」）的文件，在同 `(作者+小说名)` 组内按数字 N 排序：**数字最大者不勾选**，其余勾选（强制勾选，覆盖规则③A 对中文进度的保护）；但被勾选的文件若恰为本组 `(作者+小说名)` 内**文件大小最大者**，则也不勾选。 |

> 最终勾选集 = (规则①/② 勾选集 − 规则①/③/④ 保护集) ∪ 规则①/③ 强制勾选集。
> 规则①的精确重复(=同尺寸)不可能同时是「唯一最大文件」，故强制勾选不会与规则④冲突。

### 6.1 举例

**例一（规则① 完全相等去重）**
同一文库内《斗破苍穹》作者「天蚕土豆」有 3 个文件，名称/大小/进度完全一致，创建时间分别为 A(最早)、B、C(最新)：
- 结果：C 不勾选，A、B 勾选（删旧留新）。

**例二（规则② 纯数字进度对比）**
同书 3 个文件进度分别为 `12%`、`30%`、`30%`(都是纯数字)：
- 结果：两个 `30%` 并列最大 → 不勾选；`12%` 勾选（删低进度、留高进度）。

**例三（规则③ 含中文进度保护 + 完结特例）**

  - 完结+N番外 组合（规则⑤）：F1 `完结+3番外`(2.0MB)、F2 `完结+5番外`(5.0MB)、F3 `完结`(6.0MB) → M={F1,F2} 按 N 排序，数字最大 F2 不勾选，F1 勾选（除非 F1 恰为本组最大文件）；F3 含中文受规则③保护不勾选。最终 F1 勾选、F2/F3 保留。
同书 3 个文件：
- F1：文件名 `斗破苍穹.txt`，进度 `80%`，大小 2.0 MB
- F2：文件名 `斗破苍穹完结版.txt`，进度 `完结`，大小 5.0 MB
- F3：文件名 `斗破苍穹.txt`，进度 `完结`，大小 6.0 MB

分析：含中文进度 F2/F3 受规则③A 保护不勾选；进度最大的 F1(80%) 比所有含中文文件都小，且组内有「完结」关键词 → 触发规则③B，F1 **强制勾选**。
- 结果：F1 勾选(删)，F2、F3 不勾选(保留更完整的完结版)。

**例四（规则④ 最大文件保护）**
同书 4 个文件，进度均为纯数字 `50%`、`50%`、`50%`、`50%`，大小分别为 1.0 / 2.0 / 2.0 / 4.0 MB：
- 规则②进度并列最大 → 全部不勾选；规则④唯一最大 4.0 MB 即便被勾选也不删。
- 结果：本例无低进度项，**不勾选任何文件**。

**例五（规则④ 并列最大不保护）**
同书 3 个文件，进度 `10%`、`10%`、`10%`，大小均为 3.0 MB(完全相等)：
- 规则①五字段完全相等 → 非最新 2 本勾选、最新 1 本不勾选；规则④最大尺寸有 3 本并列 → 不据此保护。
- 结果：非最新 2 本勾选(删)，最新 1 本不勾选。

- **按书名/作者相同标记**：给同名 / 同作者文件批量加星标。
- **清除标记**：取消本文库全部标记。

## 7. 选择并删除

勾选文件（或经「勾选重复」自动勾选）后，底部出现「删除选中」：
- 点击进入「确认删除」页，分页列出待删文件，可逐项取消勾选。
- 确认后由**前台服务**逐个删除并实时显示进度与成功 / 失败数；失败项（被移动 / 权限不足）会列出。

## 8. 扫描配置

- 新增配置：名称（可选）、扫描文件夹、文件类型（如 `txt,md`）、是否递归子目录、排除文件夹。
- 从历史配置一键开始扫描。

## 9. 关键词替换（文件名数据清洗）

「设置 → 关键词替换」：维护一组规则，按顺序把「查找内容」精确替换为「替换为」（留空=删除）。分两个阶段作用：
- **扫描阶段**：清洗文件名。
- **解析阶段**：清洗解析出的书名 / 作者 / 进度 / 来源。
每条规则可启用 / 停用、调顺序。

## 10. 设置

- 主题：跟随系统 / 浅色 / 深色。
- 全局字号：小 / 标准 / 大（自动适配不同手机屏幕）。
- 关键词替换（见第 9 节）。
- 导出已标记文件清单：把当前标记列表导出（便于备份待删项）。
- 清空本地数据：清空全部扫描记录（不可恢复）。
- 关于：版本与简介。

## 11. 帮助

「帮助」页提供上述流程的图文说明，随版本更新。

## 12. 调试日志（操作留痕与排查）

「设置 → 调试日志」提供应用内日志查看：

- 展示内容：logcat 连同内存中保留的**最近 1000 条**日志，关键操作（扫描、删除等）以 `[操作]` 级别留痕，可直观看到「开始删除：共 N 个」「删除完成：成功 X / 失败 Y」等。
- 操作按钮：
  - **刷新**：重新读取最新日志；
  - **复制全部**：一键复制到剪贴板，贴给开发者 / 运维；
  - **清空**：清空本机持久化 `debug.log`（同时清内存，不影响 logcat）。
- 日志同时持久化在应用私有目录 `files/logs/debug.log`，可经 `run-as` 导出进一步分析。
