# 文件清理助手 · APP 端（Android）安装部署手册

适用于把本工程打包成 Android APK，安装到手机 / 平板 / 模拟器上使用的场景。

## 1. 环境要求

| 项目 | 要求 |
| --- | --- |
| 操作系统 | Windows / macOS / Linux 均可（构建机） |
| JDK | 17 及以上（Android Gradle Plugin 所需） |
| Android SDK | Platform 34+，Build-Tools 34+ |
| 构建工具 | 工程已内置 Gradle Wrapper（`gradlew` / `gradlew.bat`），无需另装 |
| 目标设备 | Android 8.0（API 26）及以上；调试用机需开启「开发者选项 → USB 调试」 |

> 本机已验证环境：JDK 21、Android Studio SDK（`D:/Install/Android Studio/SDK`）、MuMu 模拟器（端口 `7555`）。

## 2. 构建 APK

打开终端，进入 `android_app` 目录：

```bash
cd android_app
# Windows
.\gradlew.bat assembleDebug      # 调试包
# macOS / Linux
./gradlew assembleDebug
```

- 调试包产物：`app/build/outputs/apk/debug/app-debug.apk`
- 包名：`com.filescanner.app.debug`

发布（Release）包：

```bash
.\gradlew.bat assembleRelease
```

Release 需要签名配置。在 `android_app/app/build.gradle.kts` 的 `android { signingConfigs { ... } }` 中填入你的 keystore；或在 `gradle.properties` 配置：

```properties
FILESCANNER_STORE_FILE=../release.keystore
FILESCANNER_STORE_PASSWORD=******
FILESCANNER_KEY_ALIAS=release
FILESCANNER_KEY_PASSWORD=******
```

产物：`app/build/outputs/apk/release/app-release.apk`，包名 `com.filescanner.app`。

## 3. 安装到设备

### 3.1 真机（USB）

```bash
adb devices                       # 确认设备已识别
adb install -r app-debug.apk     # 覆盖安装调试包
# 或发布包
adb install -r app-release.apk
```

### 3.2 模拟器（以 MuMu 为例）

MuMu 默认 adb 端口为 `7555`（注意：不是 16384）。本机 adb 已复制到 `D:\adb_tools\adb.exe` 以绕过中文路径问题：

```bash
D:\adb_tools\adb.exe connect 127.0.0.1:7555
D:\adb_tools\adb.exe install -r app-debug.apk
```

若同一台电脑连接了多个设备，用 `-s <序列号>` 指定目标：

```bash
D:\adb_tools\adb.exe -s 127.0.0.1:7555 install -r app-debug.apk
```

## 4. 首次启动与权限

1. 桌面 / 应用列表点击「文件清理助手」。
2. 首页点击「开始扫描」→ 进入「扫描配置」列表 → 新增配置并「选择文件夹」。
3. 系统弹出 **SAF（文档树）授权** 对话框，请勾选「始终允许访问」后确认。应用**仅**可访问你授权的文件夹，不会读取其他目录，也**不上传**任何文件。
4. 授权后开始扫描，结果写入本机 Room 数据库。

## 5. 数据存储位置

- 数据库：`/data/data/<包名>/databases/file_scanner.db`（Room，数据库名 `file_scanner.db`）。
- 当前数据库版本：7；升级时若架构变化走 `MIGRATION_6_7`（清理旧索引），并配置 `fallbackToDestructiveMigration()` 兜底，避免升级闪退。

## 6. 卸载

直接长按卸载，或在设置中「应用 → 存储 → 清除数据 / 卸载」。卸载会删除本机数据库；如需保留请先在「设置 → 导出已标记文件清单」备份。
