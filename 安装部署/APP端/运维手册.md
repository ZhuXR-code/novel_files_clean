# 文件清理助手 · APP 端（Android）运维手册

面向实施 / 运维人员，覆盖日志、数据库、升级、常见故障。

## 1. 版本与基本信息

- 应用名：文件清理助手
- 调试包名：`com.filescanner.app.debug`
- 发布包名：`com.filescanner.app`
- 版本查看：应用内「设置 → 关于」（当前 `v1.0.0`）
- 数据库（Room）：`file_scanner.db`，版本 7

## 2. 日志

应用通过 Android `Log` 输出，tag 含 `FileScanner`。抓取：

```bash
adb logcat -s FileScanner          # 仅看本应用
adb logcat | findstr FileScanner  # Windows 过滤
# 抓崩溃：先清缓存，再启动，几秒后导出
adb logcat -c
adb shell am start -n com.filescanner.app.debug/com.filescanner.app.MainActivity
adb logcat -d > crash.log
```

崩溃关键字：`FATAL EXCEPTION`、`AndroidRuntime`、`Exception`。

### 2.1 应用内调试日志（LogUtil）

应用所有关键节点通过统一日志工具 `LogUtil` 输出，三处留痕：

1. **logcat**：`android.util.Log` 输出，tag 含 `FileScanner`，开发期 `adb logcat -s FileScanner` 可直接看。
2. **内存**：保留最近 1000 条，供「设置 → 调试日志」页面展示。
3. **持久化文件**：`/data/data/<包名>/files/logs/debug.log`（追加写入，应用私有目录）。

其中删除、扫描等关键操作会打 `[操作]` 级别的留痕（如「开始删除：共 N 个文件」「删除完成：成功 X / 失败 Y」），便于在「设置 → 调试日志」里回看用户做了什么。

> 应用内「调试日志」页面支持：查看、刷新、一键复制、清空（仅清本机 `debug.log`，不影响 logcat）。

## 3. 数据库维护

> 调试包可用 `run-as` 直接读取私有数据；发布包需 root 或先 `adb root`。

### 3.1 导出数据库（调试包）

```bash
adb shell "run-as com.filescanner.app.debug cat /data/data/com.filescanner.app.debug/databases/file_scanner.db" > file_scanner.db
```

> 注意：用 `run-as ... cat > 设备内可写路径`（如 `/data/local/tmp/`），再 `adb pull` 拉取，避免 PowerShell 重定向把二进制库编码成 UTF-16 导致损坏。

### 3.2 清空数据

- 应用内：「设置 → 清空本地数据」（不可恢复，会删除全部扫描记录）。
- 或命令行：`adb shell pm clear com.filescanner.app.debug`。

### 3.3 主要表（参考）

- `scan_run`：每次扫描（文库）记录。
- `scanned_file`：解析后的文件明细（书名/作者/进度/来源/内容指纹等）。
- `keyword_replace_rules`：关键词替换（文件名清洗）规则。
- `scan_config`：扫描配置。

## 4. 升级与迁移

- 数据库升级走 Room Migration；当前 `6 → 7` 负责清理模拟器残留的 `index_scanned_file_content_hash` 索引并对齐 `progress/source` 默认值。
- 已开启 `fallbackToDestructiveMigration()`：当本地库架构与实体不一致且无可匹配迁移时，会**销毁重建**（数据清空）。如生产环境出现此情况，通常是旧库残留导致，需先备份再清空重扫。
- 升级方式：重新 `adb install -r` 新 APK 即可，无需手动迁移。

## 5. 常见故障

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| 启动即闪退 | 旧库架构不匹配 | 清数据（`pm clear`）后重扫；确认已升级到含 MIGRATION_6_7 的版本 |
| 扫描无反应 | 未授予 SAF 文件夹权限 | 重新「开始扫描 → 选择文件夹」并勾选「始终允许」 |
| 删除无效 | 文件被占用 / 权限不足 | 查看删除确认页的失败列表；确认 SA F 授权未撤销 |
| 列表空白 | 所选文件夹无 txt/md 或筛选条件过严 | 检查文件类型设置、合集设置的数量区间 |
| 分页不生效 | 每页条数超出 10~2000 | 在分页条「每页」输入 10~2000 之间的值再点「应用」 |

## 6. 性能与容量

- 已针对 10w 级大库做真·页码分页（LIMIT/OFFSET），单页只查一页，翻页不重加载全量。
- 合集（按书名聚合）同样分页；「标记重复」在设备端计算，大库可能耗时数秒，期间显示进度提示。
