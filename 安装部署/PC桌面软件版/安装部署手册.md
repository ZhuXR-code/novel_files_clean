# 文件清理助手 · PC 桌面软件版 安装部署手册

桌面软件版 = 一个独立的 Windows 窗口应用（`FileScanner.exe`），内部自带后端与前端，使用**内置 SQLite 数据库**，**无需安装 MySQL、无需浏览器、不联网**。适用于单机个人使用、开箱即用。

> 技术形态：pywebview（系统自带 Edge WebView2 内核）+ 后台内置 uvicorn 服务（仅监听 `127.0.0.1`）。启动前已强制 `DB_BACKEND=sqlite`，网页版的 MySQL 代码路径完全不被触碰。

## 1. 面向最终用户：用安装包安装（推荐）

最简单的方式，普通用户无需安装 Python。

1. 拿到安装包 `FileScanner-Setup.exe`（见工程 `output/FileScanner-Setup.exe`，或由开发者提供）。
2. 双击运行，按向导下一步：
   - 可选择安装目录（默认 `C:\Users\<用户名>\AppData\Local\Programs\文件扫描管理系统（本地版）`，无需管理员即可安装到该可写目录；也可改为任意路径，如 `D:\Tools\...`）；
   - 可勾选「创建桌面快捷方式」。
3. 安装完成后从桌面 / 开始菜单启动「文件扫描管理系统（本地版）」。

**运行环境要求：** Windows 10/11（x64）。系统一般已自带 Microsoft Edge WebView2 运行时；极少数精简系统若窗口打不开，安装一次微软官方的「WebView2 Runtime」即可。

### 数据存放位置

- 数据库默认存放在**安装目录内**的 `FileScannerData\file_scanner.db`（例如
  `C:\Users\<用户名>\AppData\Local\Programs\文件扫描管理系统（本地版）\FileScannerData\file_scanner.db`）。
- 即数据随安装目录走：用户在安装向导中指定的路径即为数据根目录，
  卸载/移动整个安装目录即可整体带走或重置，不再散落到 `%APPDATA%`。
- 可用环境变量 `SQLITE_DB_PATH` 显式指定其它数据目录（便于自定义）。

### 卸载

从「设置 → 应用」或开始菜单卸载。卸载时会弹窗询问是否同时删除用户数据
（安装目录内的 `FileScannerData`）；选择「是」则彻底清除数据库、下次安装为
全新空白状态，选择「否」则保留数据。覆盖安装 / 升级时会自动保留数据库与数据。

## 2. 面向开发者：源码直接运行

```bash
# 进入工程根目录
cd <工程根>/txt文件清理-单工程清理

# 安装依赖（含 pywebview / pythonnet 等桌面版依赖）
pip install -r requirements.txt

# 启动桌面软件版（自动以 SQLite 模式打开独立窗口）
python launcher.py

# 便携模式：数据库放程序旁 FileScannerData\
python launcher.py --portable
```

`launcher.py` 会：① 设 `DB_BACKEND=sqlite`；② 从 8000 起找一个空闲端口，在后台线程起 uvicorn（仅 `127.0.0.1`）；③ 服务就绪后用 pywebview 打开窗口嵌入前端；④ 关闭窗口即退出整个进程。

> 关键依赖：`pywebview`、`pythonnet`、`clr_loader`（Windows 上 WebView2 需经 .NET 桥接）。缺少 `clr` 会报 `No module named 'clr'`。

## 3. 面向开发者：打包成 EXE 与安装包

### 3.1 生成单文件 EXE（PyInstaller）

```bash
# 打包前先结束残留的 FileScanner 进程，避免 PermissionError
python -m PyInstaller build/FileScanner.spec --noconfirm
```

- 产物：`dist/FileScanner.exe`（单文件，已内置前端与全部依赖，`console=False` 无黑窗）。
- spec 要点：把 `frontend/` 作为数据文件打入；用 `collect_submodules` 收集 `webview` / `pythonnet` / `clr_loader`；排除 `matplotlib/numpy/pandas` 减小体积；**切勿 exclude pythonnet/clr**。

### 3.2 生成 Windows 安装向导（Inno Setup）

```bash
# 在 build 目录下执行
cd build
ISCC.exe FileScanner.iss
```

- 产物：`build/output/FileScanner-Setup.exe`。
- iss 要点：`SourceDir=.`（即 `build\`），`[Files]` 源用 `../dist/FileScanner.exe`；中文界面需 `compiler:Languages\ChineseSimplified.isl`；**`Compression=none`**（单文件 EXE 已压缩，用 lzma2 重压极慢且几乎无收益）。

## 4. 常见安装问题

| 现象 | 原因 | 处理 |
| --- | --- | --- |
| 双击 EXE 无窗口 / 闪退 | 缺 WebView2 运行时 | 安装微软官方 WebView2 Runtime |
| 源码运行报 `No module named 'clr'` | 缺 pythonnet | `pip install pywebview pythonnet` |
| 打包报 `PermissionError` | 残留进程占用 exe | 任务管理器结束 FileScanner 进程后重试 |
| 打包后窗口白屏 | 前端未打入 / 收集不全 | 确认 spec 已 `collect_data_files('webview')` 且打入 `frontend/` |
| 安装包编译很慢 / 超时 | 用了 lzma2 重压 | 改 `Compression=none` |
