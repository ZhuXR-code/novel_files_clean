# 文件清理助手 · PC 桌面软件版 运维手册

## 1. 基本信息

- 形态：Windows 独立窗口应用 `FileScanner.exe`（pywebview + WebView2 内核）
- 后端：内置 uvicorn，运行 `backend.app:app`，仅监听 `127.0.0.1`
- 端口：从 **8000** 起自动探测空闲端口（本机同时跑网页版也不冲突）
- 数据库：**内置 SQLite**
  - 默认：`%APPDATA%\FileScanner\file_scanner.db`
  - 便携：`FileScanner.exe --portable` → 程序旁 `FileScannerData\file_scanner.db`
  - 亦可用环境变量 `SQLITE_DB_PATH` 指定绝对路径
- 关键环境变量：`DB_BACKEND=sqlite`（launcher 已自动设置）、`SQLITE_DB_PATH`（可选）

## 2. 日志

- PC 端内置日志模块（`backend/logger.py`），关键节点落盘工程 `logs/app.log`（源码运行时）并支持轮转。
- EXE 运行时（`console=False`）无控制台窗口；排障时可临时用源码方式 `python launcher.py` 在终端观察实时日志。
- 关键词搜索：`ERROR`、`Warning`、`数据库`、`本地服务`、`端口`。

## 3. 数据库维护（SQLite）

SQLite 是单文件数据库，维护极简：

- **备份**：直接复制 `file_scanner.db` 文件即可（建议在软件关闭状态下复制）。
- **恢复**：用备份文件覆盖同路径的 `file_scanner.db`。
- **迁移换机**：拷贝该 `.db` 文件到新机对应目录；便携模式则整目录拷走。
- 可用 DB Browser for SQLite 等工具打开查看。

主要表（与网页版 / APP 端一致）：

- `scan_run`：每次扫描（文库）记录。
- `scanned_file`：解析后的文件明细。
- `scan_config`：扫描配置。
- `keyword_replace_rules`：关键词替换（文件名清洗）规则。

> 表结构以 `backend/models.py` 与启动时自动迁移为准，不要手工大改结构。

## 4. 升级

- **安装包用户**：直接运行新版 `FileScanner-Setup.exe` 覆盖安装即可，`%APPDATA%\FileScanner\` 下的数据库自动保留、自动兼容迁移。
- **开发者重新打包**：
  ```bash
  # 1. 先结束残留 FileScanner 进程，避免 PermissionError
  # 2. 重新打包 EXE
  python -m PyInstaller build/FileScanner.spec --noconfirm
  # 3. 重新生成安装包
  cd build && ISCC.exe FileScanner.iss
  ```

## 5. 常见故障

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| 双击无窗口 / 闪退 | 缺 Edge WebView2 运行时 | 安装微软官方 WebView2 Runtime |
| 窗口白屏 | 前端未打入 / webview 资源收集不全 | 重新按 spec 打包（含 `collect_data_files('webview')`、打入 `frontend/`） |
| 启动弹「本地服务未能在 30 秒内启动」 | 后端起不来（依赖缺失 / 端口全被占） | 源码方式跑 `python launcher.py` 看终端日志定位 |
| 源码运行报 `No module named 'clr'` | 缺 pythonnet | `pip install pywebview pythonnet` |
| 打包报 `PermissionError` | exe 被残留进程占用 | 结束 FileScanner 进程后重试 |
| 打包 / 编译安装包极慢 | Inno Setup 用了 lzma2 重压 | 改 `Compression=none` |
| 数据不见了 | 换了默认/便携模式，读的是不同 .db | 确认启动参数是否带 `--portable`，核对数据库路径 |

## 6. 安全与容量

- 后端只绑定 `127.0.0.1`，天然不对外网暴露，无需额外防火墙配置。
- 全部数据在本机 SQLite 单文件内，隐私可控；定期复制 `.db` 备份即可。
- 大库（十万级）已做分页查询，前端「每页条数」默认 100、范围 10~2000，「跳至页」可直达。
- 删除操作作用于本机真实磁盘文件，请提醒用户删除前先「导出已标记清单」备份。
